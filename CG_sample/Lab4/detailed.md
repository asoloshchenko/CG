# Лаб 4 «Текстурирование» — подробный разбор для защиты

Стек: **WebGL2** + JS, `gl-matrix`. Файл — [main.js](main.js).

Главная задача: показать **многослойное смешение текстур** — на одном кубе сразу работают цвет, текстура материала и текстура цифры с альфа-каналом, причём вклад каждого слоя регулируется в реальном времени.

---

## 0. Что такое текстура и UV

- **Текстура** — двумерный массив пикселей (texel'ов) в видеопамяти. В шейдере читается через `sampler2D` функцией `texture(sampler, uv)`.
- **UV-координаты** — пара чисел `(u, v) ∈ [0, 1]` на каждую вершину, говорит «в какую точку текстуры ткнуть». Между вершинами треугольника UV **интерполируются**, и каждый пиксель экрана берёт свой texel из текстуры.
- **Альфа-канал (A в RGBA)** — степень непрозрачности. Здесь используется как **маска**: цифра нарисована на прозрачном фоне, и где `a = 0`, цифра не отображается.

## 1. Инициализация и состояние (main.js:1–37)

## 2. Два пайплайна — две пары шейдеров

В лабе сосуществуют **две независимые программы**, потому что у кубов и у моделей принципиально разные данные.

### 2a. Шейдер кубов — `vsSource` / `fsSource` (main.js:42, 61)

**Вершинный** просто прокидывает UV и базовый цвет:
```glsl
gl_Position = uProjection * uModel * vec4(aPosition, 1.0);
vColor = uBaseColor;
vUV = aUV;
```

**Фрагментный** — самый важный код лабы:
```glsl
vec3 tex1 = texture(uTextureMat, vUV).rgb;                       // материал
vec4 tex2 = texture(uTextureNum, vec2(vUV.x, 1.0 - vUV.y));      // цифра (флипнут V)

float numberMask = clamp(uNumWeight, 0.0, 1.0) * tex2.a;         // альфа цифры × вес
vec3 resTex      = mix(tex1, tex2.rgb, numberMask);              // материал ⨯ цифра
vec3 resColor    = vColor * resTex;                              // подкрашивание
vec3 finalColor  = mix(resTex, resColor, clamp(ucolorWeight,0.,1.));
```

Разбор слой за слоем:
1. `tex1` — диффузная текстура материала (золото / серебро / бронза).
2. `tex2` — RGBA текстура цифры на прозрачном фоне; V переворачивается, потому что PNG в WebGL грузится «вверх ногами».
3. `numberMask = uNumWeight · alpha` — управляемый множитель: 0 → цифры нет, 1 → цифра полностью перекрывает материал в своих пикселях.
4. `mix(a, b, t) = a·(1−t) + b·t` — линейная интерполяция: между «чистый материал» и «материал + цифра».
5. `vColor * resTex` — умножение на базовый цвет: **тонирует** текстуру, не заменяя её. Если цвет белый — текстура остаётся как есть, если красный — окрашивает в красный.
6. Финальный `mix(resTex, resColor, ucolorWeight)` — плавно перетекает от «без окраски» к «с окраской».

Это и есть выполнение пункта задания: **3 источника (цвет + материал + цифра), каждый со своим весом**.

### 2b. Шейдер моделей — `vsModel` / `fsModel` (main.js:88, 100)

Минимум: одна диффузная текстура, никаких смешиваний. Просто `texture(uTexture, vUV)`.

## 3. `createShader` и линковка программ (main.js:110–147)

Стандартный паттерн: `createShader → shaderSource → compileShader → проверка COMPILE_STATUS`; затем `createProgram → attachShader (×2) → linkProgram → проверка LINK_STATUS`. Делается дважды — для `program` (кубы) и `programModel` (модели). Здесь `bindAttribLocation` не используется: атрибутные слоты потом достаются через `getAttribLocation` под каждый VAO отдельно.

## 4. Геометрия куба (main.js:156–227)

Жёстко прописанный массив из **24 вершин** (по 4 на каждую из 6 граней, нельзя переиспользовать вершины разных граней — у них разные UV и нормали должны быть). Формат вершины — **8 float**: `pos(3) + color(3) + uv(2)`. Индексы — `Uint16Array`, по 2 треугольника на грань.

VAO:
- Создаём VAO, биндим.
- VBO с `STATIC_DRAW`.
- EBO с индексами.
- Три атрибута через `vertexAttribPointer` (stride = 32 байта):
  - `aPosition` — offset 0, 3 float,
  - `aColor` — offset 12, 3 float,
  - `aUV` — offset 24, 2 float.
- `bindVertexArray(null)` — отвязываем, чтобы дальнейшие операции случайно его не испортили.

## 5. `loadTexture(src)` (main.js:239)

```js
const texture = gl.createTexture();
gl.bindTexture(gl.TEXTURE_2D, texture);
gl.texParameteri(..., gl.TEXTURE_WRAP_S, gl.CLAMP_TO_EDGE);
gl.texParameteri(..., gl.TEXTURE_WRAP_T, gl.CLAMP_TO_EDGE);
gl.texParameteri(..., gl.TEXTURE_MIN_FILTER, gl.LINEAR);

const img = new Image();
img.src = src;
img.onload = () => {
  gl.bindTexture(gl.TEXTURE_2D, texture);
  gl.texImage2D(gl.TEXTURE_2D, 0, gl.RGBA, gl.RGBA, gl.UNSIGNED_BYTE, img);
};
return texture;
```

Что важно объяснить:
- **`createTexture` возвращается сразу**, а данные подгружаются асинхронно в `onload` — до этого момента текстура «пустая», и куб первые кадры может быть чёрным.
- **`TEXTURE_WRAP_S/T = CLAMP_TO_EDGE`** — что делать, если UV выходит за `[0,1]`. Варианты: `REPEAT` (тайлинг), `MIRRORED_REPEAT` (отражение), `CLAMP_TO_EDGE` (тянем крайний пиксель). Здесь clamp — потому что текстура цифры по краям прозрачная и тайлить её бессмысленно.
- **`TEXTURE_MIN_FILTER = LINEAR`** — как фильтровать при уменьшении (камера далеко). `NEAREST` — пиксельно, `LINEAR` — билинейно (среднее 4 соседей). Для красивого результата лучше `LINEAR_MIPMAP_LINEAR`, но тогда нужно `gl.generateMipmap()`.
- **`texImage2D(target, level, internalFormat, format, type, source)`** — фактически грузит пиксели в GPU. `level=0` — основной уровень, остальные — мипмапы.

## 6. Управление вкладами (main.js:279)

```js
R / F — colorWeight ± 0.05  // вклад базового цвета
T / G — numWeight   ± 0.05  // вклад цифры
```

Никакого `setUniforms` после изменения не вызывается — uniform'ы заливаются каждый кадр внутри `renderCube`.

## 7. `loadOBJ(url)` (main.js:288) — упрощённый парсер

Отличия от Lab3:
- Читает только `v`, `vt`, `f` (без нормалей).
- **V переворачивается сразу при чтении**: `1 - parseFloat(parts[2])` — компенсирует разное соглашение об оси V между OBJ и WebGL.
- Не дедуплицирует вершины и **не использует EBO**: разворачивает каждый треугольник как 3 уникальные вершины в плоский массив. Проще, но больше памяти; рисуется через `drawArrays`.
- Нормализация bbox в `[-0.5, 0.5]` (масштаб `1/maxExtent`, у Lab3 было `2/maxExtent`).

## 8. `loadModel(name, posX, posY, scale, textureName, ext)` (main.js:367)

1. Грузит OBJ и текстуру.
2. Создаёт собственный VAO для модели с двумя атрибутами (`aPosition` + `aUV`, stride = 20).
3. Сохраняет в массив `models[]` объект `{ vao, count, texture, posX, posY, scale }`.

Загружаются три модели разных форматов (две PNG, одна JPG).

## 9. `renderCube(num, tx)` (main.js:411)

Для каждого из трёх кубов:
1. Считаем `projection` (перспектива) и `model` (поворот + масштаб 0.5 + позиция).
2. Заливаем `uModel`, `uProjection`.
3. `uBaseColor` — здесь у всех трёх кубов одинаково красный `[1,0,0]` (закомментированы изначальные «золото/серебро/бронза»).
4. **Бинд двух текстур в два разных текстурных юнита**:
   ```js
   gl.activeTexture(gl.TEXTURE0);
   gl.bindTexture(gl.TEXTURE_2D, textureMat[num]);
   gl.uniform1i(texLoc1, 0);              // sampler смотрит в юнит 0

   gl.activeTexture(gl.TEXTURE1);
   gl.bindTexture(gl.TEXTURE_2D, textureNum[num]);
   gl.uniform1i(texLoc2, 1);              // sampler смотрит в юнит 1
   ```
   Это критически важный для защиты момент: `sampler2D` в шейдере хранит **номер текстурного юнита**, а не саму текстуру. Чтобы передать две текстуры одновременно — нужны два юнита.
5. Заливаем `colorWeight`, `numWeight`.
6. `bindVertexArray(vao)` + `drawElements(TRIANGLES, 36, UNSIGNED_SHORT, 0)`.

## 10. `render()` (main.js:458)

Каждый кадр:
1. Очистка цвета и глубины.
2. `useProgram(program)` — рисуем три куба со смещениями `tx ∈ {-1, 0, 1}`.
3. Если модели загрузились — `useProgram(programModel)`, считаем общую `projection`, для каждой модели заливаем матрицу и текстуру, рисуем `drawArrays(TRIANGLES, 0, count)`.
4. Тикаем углы, обновляем HUD `#info`, следующий кадр через `requestAnimationFrame`.

## Что точно спросят на защите

| Вопрос | Ответ |
|---|---|
| Что такое UV? | Двумерные координаты `(u,v) ∈ [0,1]` на вершину, указывающие на точку текстуры. Между вершинами интерполируются, фрагмент сэмплирует текстуру по своему UV. |
| Что такое sampler2D? | GLSL-тип, который ссылается на текстурный юнит, а не на саму текстуру. В юнит привязывается конкретная текстура командой `bindTexture` после `activeTexture(TEXTUREi)`. |
| Зачем `activeTexture` + `bindTexture` + `uniform1i`? | Активируем юнит, привязываем к нему текстуру, говорим sampler'у «смотри в этот юнит». Без `activeTexture` все `bindTexture` идут в юнит 0. |
| Как работает `mix(a, b, t)`? | Линейная интерполяция `a*(1-t) + b*t`. При t=0 → a, при t=1 → b. |
| Зачем альфа-канал у текстуры цифры? | Маска прозрачности. `numberMask = uNumWeight * alpha` — гарантирует, что цифра проявляется только в своих пикселях, а не закрашивает всю грань. |
| Почему `vec2(vUV.x, 1 - vUV.y)` для второй текстуры? | Цифра в PNG записана с другой ориентацией оси V — компенсируем флипом. |
| Почему 24 вершины у куба, а не 8? | У каждой грани свои UV (и нормали, если бы были), а WebGL допускает только один общий индекс на вершину. Поэтому угловая точка дублируется в трёх гранях с разными UV. |
| Что такое stride и offset в `vertexAttribPointer`? | Stride — байтов между началами соседних вершин в VBO; offset — смещение нужного атрибута внутри вершины. Здесь stride=32, offset позиции=0, цвета=12, UV=24. |
| Что делает `CLAMP_TO_EDGE` vs `REPEAT`? | Поведение за границей `[0,1]`: clamp — берёт крайний texel; repeat — тайлинг; mirrored_repeat — зеркалит. |
| Чем `LINEAR` отличается от `NEAREST`? | NEAREST — берётся ближайший texel (пиксельно). LINEAR — билинейное среднее 4 соседних texel'ов (мягко). |
| Чем `drawElements` отличается от `drawArrays`? | drawElements использует индексы из EBO (экономия памяти, переиспользование вершин). drawArrays идёт подряд по VBO. Кубы рисуются elements, модели — arrays (т.к. парсер их сразу развернул). |
| Зачем две разные программы шейдеров? | У кубов — многослойное смешение 3 источников, у моделей — простая диффузная текстура. Делать всё одним шейдером с кучей флагов — менее читаемо и медленнее. |
| Что делает `texImage2D`? | Загружает пиксели изображения в выделенную текстуру на GPU с указанным форматом (`RGBA`, `UNSIGNED_BYTE`). |
| Почему текстуры грузятся асинхронно? | `Image.onload` — стандартный браузерный механизм; до загрузки картинки её нельзя залить в GPU. Поэтому первые кадры куб может быть чёрным/неинициализированным. |
