В корне нет Lab1/Lab2 — первая по списку **Lab3**. Стек: чистый **WebGL2** + JS, библиотека только `gl-matrix` для матриц. Файл — [main.js](main.js).

## 0. База: WebGL2, VBO, EBO, VAO, шейдеры

Чтобы понимать любой метод ниже, нужны 4 термина:

- **VBO** (Vertex Buffer Object) — буфер в видеопамяти GPU с «сырыми» данными вершин (позиции, нормали, UV).
- **EBO / IBO** (Element / Index Buffer Object) — буфер индексов, по которым GPU собирает треугольники из вершин VBO. Экономит память: одна вершина переиспользуется несколькими треугольниками.
- **VAO** (Vertex Array Object) — «конфигурация рисования». Запоминает: какой VBO привязан, какой EBO, как именно из байтов VBO нарезать атрибуты (где позиция, где нормаль, какой stride/offset). При рендере достаточно `bindVertexArray(vao)` — и драйвер сам всё восстановит, не надо каждый кадр заново настраивать `vertexAttribPointer`.
- **Шейдер** — программа на GLSL, исполняемая на GPU:
  - **Vertex shader** — выполняется для каждой вершины, обязан выставить `gl_Position` (позиция в clip-space).
  - **Fragment shader** — для каждого пикселя треугольника, обязан выставить выходной цвет.

  Между ними значения `out vec3 vNormal` (вершинный) → `in vec3 vNormal` (фрагментный) **интерполируются** растеризатором — это ключевой механизм.

## 3. `updateInfo()` (main.js:44)

## 4. `createTransformMatrix(ax,ay,az, sx,sy,sz, tx,ty,tz)` (main.js:64)

Собирает модельную матрицу через `gl-matrix`:
```
M = T(tx,ty,tz) · Rz · Ry · Rx · S(sx,sy,sz)
```
Порядок важен — в `gl-matrix` каждый `mat4.translate/rotate/scale` **домножает справа**, поэтому первая по коду операция применится последней к вершине. Логика: вершину сначала масштабируем, потом крутим вокруг X→Y→Z, потом сдвигаем в мир.

## 5. Шейдеры

### `vsPhong` / `fsPhong` (Phong shading)

Вершинный — минимум: трансформирует позицию и нормаль в мировое пространство и передаёт во фрагмент:
```glsl
vec4 worldPos = uModel * vec4(aPosition, 1.0);
vPos    = worldPos.xyz;
vNormal = normalize(mat3(uModel) * aNormal);
gl_Position = uProjection * worldPos;
```
`mat3(uModel)` отбрасывает translate — нормали поворачиваются и масштабируются, но не сдвигаются. (Строго правильно — `transpose(inverse(mat3(uModel)))`, но при равномерном масштабе результат тот же.)

Фрагментный считает освещение **на каждый пиксель**:
```
ambient    = kA · baseColor                                 // фон
L          = normalize(lightPos − vPos)                     // направление на свет
d          = distance(lightPos, vPos)
attenuation= 1 / (kc + kl·d + kq·d²)                        // затухание
diff       = max(N·L, 0)                                    // Ламберт
diffuse    = diff · baseColor · attenuation
if (uUsePhong) {
  V = normalize(−vPos);                                     // взгляд (камера в 0)
  R = reflect(−L, N);                                       // отражённый луч
  spec = pow(max(V·R, 0), 32) · attenuation;                // блик Фонга
}
color = ambient + diffuse + spec
```

### `vsGouraud` / `fsGouraud` (Gouraud shading)

**Тот же расчёт, но в вершинном шейдере.** На выходе `out vec3 vColor`, фрагментный просто пишет интерполированный цвет:
```glsl
outColor = vec4(vColor, 1.0);
```

Разница для защиты — это ключевое:
- **Phong shading** — освещение в фрагменте → гладкие блики, но в N раз дороже (N = пикселей на треугольник).
- **Gouraud shading** — освещение в вершине, цвет линейно интерполируется. Дёшево, но блик «размазывается» / пропадает на крупных треугольниках, появляются полосы Маха.

Внимание: «Phong» — это **два** разных понятия в коде.
1. **Phong shading** (где считаем) — клавиша **G/P**.
2. **Phong reflection** = добавление зеркального члена specular поверх Ламберта — клавиша **L** (флаг `uUsePhong`).

## 6. `createShader(type, src)` (main.js:168)

`createShader` → `shaderSource` → `compileShader` → проверка `COMPILE_STATUS`. Если ошибка — `getShaderInfoLog` в консоль.

## 7. `createProgramWithBindings(vsSrc, fsSrc, bindings)` (main.js:177)

1. `createProgram()` — пустая программа.
2. Прикрепляет скомпилированные вершинный и фрагментный шейдеры.
3. **`bindAttribLocation(prog, 0, "aPosition")`** — жёстко закрепляет: «атрибут с именем aPosition будет в слоте 0». Вызвать **до `linkProgram`**. Зачем — чтобы у обеих программ (Phong и Gouraud) атрибуты лежали в одинаковых слотах и один VAO работал с обеими.
4. `linkProgram` + проверка `LINK_STATUS`.

## 8. Uniform-локации и `setUniforms()` (main.js:196–233)

`gl.getUniformLocation(prog, "uName")` — узнаёт «адрес» uniform-переменной внутри программы. Делается один раз после линковки.

`setUniforms(prog, uniforms)` — заливает все текущие JS-значения (`lightPos`, `ambientPower`, …) в uniform'ы указанной программы. Вызывается каждый раз, когда меняется параметр с клавиатуры. **Важно:** uniform'ы хранятся внутри программы, поэтому при изменении значения нужно обновить **обе** программы (Phong и Gouraud), иначе при переключении G↔P отвалятся настройки.

## 9. `loadOBJ(url)` (main.js:240) — ручной парсер .obj

OBJ — текстовый формат:
- `v x y z` — позиция вершины,
- `vn x y z` — нормаль,
- `vt u v` — UV-координата,
- `f v1/vt1/vn1 v2/vt2/vn2 …` — грань (полигон), индексы 1-based.

Алгоритм:
1. Читает построчно, разбирает `v/vn/vt` в отдельные массивы `pos`, `nor`, `uv`.
2. Для каждой грани триангулирует **веером**: для полигона `[a,b,c,d]` → треугольники `(a,b,c)`, `(a,c,d)`.
3. `getIndex(v, vt, vn)` — дедупликация: одна и та же тройка индексов даёт одну вершину в финальном буфере (через `Map` по строке `"v/vt/vn"`). Это важно, потому что в OBJ позиция и нормаль индексируются раздельно, а GPU умеет только один общий индекс на вершину.
4. На выходе каждая вершина = **8 float**: `px py pz nx ny nz u v`.
5. **Нормализация**: ищет bbox модели, центрирует в (0,0,0) и масштабирует так, чтобы максимальная сторона стала 2 (то есть модель помещается в куб `[-1,1]`). Это чтобы любая модель из интернета сразу «влезала» в сцену.

## 10. `createMesh(data)` (main.js:331) — собственно VAO

```js
const vao = gl.createVertexArray();
gl.bindVertexArray(vao);

const vbo = gl.createBuffer();
gl.bindBuffer(gl.ARRAY_BUFFER, vbo);
gl.bufferData(..., data.vertices, gl.STATIC_DRAW);   // загрузили вершины в GPU

const ebo = gl.createBuffer();
gl.bindBuffer(gl.ELEMENT_ARRAY_BUFFER, ebo);
gl.bufferData(..., data.indices, gl.STATIC_DRAW);    // загрузили индексы

const stride = 8 * 4;                                // 8 float по 4 байта
gl.enableVertexAttribArray(0);
gl.vertexAttribPointer(0, 3, gl.FLOAT, false, stride, 0);      // позиция
gl.enableVertexAttribArray(1);
gl.vertexAttribPointer(1, 3, gl.FLOAT, false, stride, 3 * 4);  // нормаль
```

`vertexAttribPointer(index, size, type, normalize, stride, offset)` — объясняет GPU «как читать буфер»:
- `index` — слот атрибута (тот, что закрепили `bindAttribLocation`),
- `size` — сколько компонент (3 для vec3),
- `stride` — шаг между вершинами в байтах (32),
- `offset` — где внутри вершины лежит этот атрибут (позиция — 0, нормаль — 12, UV — 24).

**`STATIC_DRAW`** — подсказка драйверу: «данные не меняются, оптимизируй под это».

`gl.UNSIGNED_INT` для индексов — потому что моделей много вершин, `Uint16` (max 65535) не хватит. В WebGL2 это работает из коробки, в WebGL1 нужно расширение.

## 11. `init()` (main.js:365)

Асинхронно грузит три OBJ (снеговик, лавочка, голова), создаёт для каждого меш + сохраняет позицию `tx,ty,tz` и масштаб. После загрузки запускает первый кадр.

## 12. Обработчик клавиш (main.js:408)

- **R** — пауза вращения (`isRotating`),
- **+ / −** — `ambientPower ± 0.02`,
- **L** — переключение `usePhongModel` (specular on/off),
- **G / P** — смена активной программы (Gouraud / Phong shading),
- **i / Shift+i** — `attLinear ±`,
- **o / Shift+o** — `attQuadratic ±`.

После каждого изменения вызывается `setUniforms` для **обеих** программ + `updateInfo()` для HUD.

## 13. `render()` (main.js:487) — цикл

Каждый кадр:
1. `gl.clear(COLOR_BUFFER_BIT | DEPTH_BUFFER_BIT)` — очищаем экран и z-буфер.
2. `mat4.perspective(projection, π/4, aspect, 0.1, 100)` — стандартная проекционная матрица: FOV 45°, ближняя плоскость 0.1, дальняя 100. Загружаем в uniform `uProjection`.
3. Для каждого объекта: считаем `M = T·R·S`, заливаем в `uModel`, биндим его VAO, вызываем `drawElements(TRIANGLES, count, UNSIGNED_INT, 0)` — GPU читает индексы из EBO, по ним берёт вершины из VBO, прогоняет через шейдер, растеризует.
4. Если `isRotating` — инкрементим углы.
5. `requestAnimationFrame(render)` — следующий кадр (синхронизировано с монитором, обычно 60 FPS).

## Что точно спросят на защите и краткие ответы

| Вопрос | Ответ |
|---|---|
| Что такое VAO и зачем? | Объект, который запоминает связку VBO+EBO+layout атрибутов. При рендере один `bindVertexArray` восстанавливает всю настройку. |
| Чем Gouraud отличается от Phong shading? | Gouraud считает свет в вершине, фрагмент получает интерполированный цвет — дёшево, плохие блики. Phong считает свет в фрагменте, интерполируются нормаль и позиция — дороже, качественные блики. |
| Что такое модель Ламберта? | Диффузная составляющая: `I = max(N·L, 0) · color`. Зависит только от угла между нормалью и направлением на свет, не зависит от позиции наблюдателя. |
| Что такое блик Фонга (specular)? | `spec = pow(max(V·R, 0), n)`, где V — на камеру, R — отражённый луч света, n — «глянцевость» (здесь 32). Зависит от наблюдателя — поэтому при Gouraud блик плохой. |
| Зачем три коэффициента затухания? | Физически свет затухает как `1/d²`. Формула `1/(kc+kl·d+kq·d²)` — артистический контроль: kc убирает деление на ноль вблизи источника, kl даёт мягкое затухание, kq — реалистичное квадратичное. |
| Почему нормаль умножается на `mat3(uModel)`? | Чтобы повернуть/масштабировать нормаль без translate. Строго правильно — `transpose(inverse(mat3(M)))`, но при равномерном масштабе совпадает. |
| Зачем `bindAttribLocation` до линковки? | Жёстко фиксирует слот атрибута. Без него драйвер сам назначает индексы — у двух программ могли бы оказаться разные, и один VAO с двумя программами не работал бы. |
| Зачем тест глубины? | Чтобы пиксель ближнего треугольника не перекрывался дальним. Сравнение по z-буферу. |
| Что такое uniform vs in/out? | uniform — одинаковое значение для всех вершин/фрагментов в одном draw call (матрицы, позиция света). `in/out` — атрибут на вершину или интерполированное значение во фрагмент. |
| Почему очищаем оба буфера? | COLOR — стереть прошлый кадр, DEPTH — обнулить z-буфер, иначе следующий кадр будет сравнивать с устаревшими глубинами. |
