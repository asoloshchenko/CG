# Лаб 5 — Bump mapping и Normal mapping. Как это сделано

Стек: **WebGL1** + JS (используется `texture2D`, `varying`, GLSL ES 1.00). Вся логика в [main.js](main.js).

## Две модели и два режима
В `models` (main.js:577) хранятся два «пресета»:
- **orange** — сфера + диффузка апельсина + height-карта (`food_0022_color_1k.jpg`, `food_0022_ao_1k.jpg`). Флаг `useNormalMap = 0.0` → используется **bump mapping** через разностный градиент карты высот.
- **cube** — куб + диффузка + готовая normal-карта (`DefaultMaterial_normal.png`). Флаг `useNormalMap = 1.0` → используется **normal mapping**.

Переключение клавишами **1 / 2** (main.js:719). Сила деталей `detailStrength` крутится стрелками **↑ / ↓** (main.js:723–727), границы свои для каждой модели (`minStrength..maxStrength`).

## Что считает фрагментный шейдер (main.js:35)
В шейдер приходят: нормаль, касательная (tangent), UV, мировая позиция.

1. Строится **базис TBN** прямо во фрагменте:
   ```glsl
   tangent   = normalize(vTangent - normal * dot(vTangent, normal)); // Грэма–Шмидта
   bitangent = cross(normal, tangent);
   ```
2. Получает «локальную» нормаль в tangent-space:
   - **Normal map** (`uUseNormalMap > 0.5`): просто `texture2D(uNormalMap, vUV).rgb * 2 - 1`, xy масштабируется силой.
   - **Bump map** (иначе): берётся высота в текущем тексели `hC`, в соседних `hU`, `hV` (сдвиги задаются uniform-ом `uHeightTexel = 1/1024`). Разностные градиенты → нормаль `(dHdU, dHdV, 1)`.
3. Перевод обратно в мировое пространство: `N_world = T·n.x + B·n.y + N·n.z`.
4. Освещение **Фонга**: ambient + diffuse (`max(N·L,0)`) + specular (`pow(max(V·R,0), uShininess)`), `uShininess = 48`.

## Касательные считаются на CPU
Раз модели — обычные OBJ, тангенсов в них нет. `buildTangents()` (main.js:403) для каждого треугольника решает систему по дифференциалам UV:

```
det = dUV1.x * dUV2.y - dUV1.y * dUV2.x
T   = (dUV2.y * e1 - dUV1.y * e2) / det
```

Тангенсы по вершинам аккумулируются, потом ортогонализуются к нормали и нормируются. Если вырождение (длина ≈ 0) — берётся запасной вектор.

## Парсер OBJ
`parseOBJ()` (main.js:327) поддерживает `v/vt/vn`, отрицательные индексы, авто-генерацию UV для сферы (через `atan2`/`asin` по позиции, main.js:318) — это спасает, если у меша нет `vt`.

## Камера
Не двигается: `lookAt([0, 0.3, 3.2], 0, [0,1,0])` собирается один раз. Объект медленно крутится вокруг Y (`t * 0.1` в `render()`, main.js:733).

## UI
Строка `#info` обновляется в `updateInfoText()` (main.js:667) — показывает активную модель, текущую интенсивность и диапазон.

## Что соответствует пунктам задания
1. **Bump mapping на апельсине** — сфера с диффузкой апельсина и карта высот, нормаль восстанавливается из градиента яркости.
2. **Normal mapping** — куб с готовой normal-картой в tangent-space.
3. **Освещение Фонга** — ambient + diffuse + specular с `pow(..., 48)`.
