# Лаб 7 «Постобработка» — подробный разбор для защиты

Стек: **WebGL2** + JS. Файл — [main.js](main.js).

Главная задача: реализовать **постобработку** — сцена рисуется не сразу на экран, а в **off-screen framebuffer**, потом весь экран закрашивается одним полноэкранным треугольником через постшейдер, который читает текстуру сцены и накладывает эффекты: **Bloom, Vignette, Grain (шум), Color Grading через 3D LUT**.

---

## 0. Базовые понятия

- **Framebuffer (FBO)** — «холст» для рендера. Default framebuffer — это экран. Можно создать свой и рисовать туда — результат окажется в текстуре, которую дальше можно использовать как обычную.
- **Off-screen rendering** — рендер не в экран, а в FBO. Базовая техника для постобработки, отражений, теней.
- **Постобработка** — рисуем сцену в FBO → берём её как текстуру → рисуем полноэкранный квадрат с шейдером, который читает эту текстуру и модифицирует пиксели → результат на экран.
- **Полноэкранный квад (fullscreen quad)** — два треугольника, покрывающие весь экран в clip-space `[-1, 1]²`. Используется как «холст» для постшейдера.

**Пайплайн в одну строку:**
```
[сцена] → sceneFbo(color + depth) → [postShader, fullscreen quad] → screen
```

## 3. Постшейдер (main.js:199–287) — ядро лабы

### Вершинный (`postVsSource`)
```glsl
in vec2 aPosition;
in vec2 aUV;
out vec2 vUV;
void main() {
  vUV = aUV;
  gl_Position = vec4(aPosition, 0.0, 1.0);
}
```
Никаких матриц — координаты вершин уже в clip-space `[-1,1]`. Это и есть **полноэкранный квад**.

### Фрагментный — все эффекты в одном

```glsl
vec3 base = texture(uSceneTex, vUV).rgb;
```
Берём пиксель сцены, дальше последовательно применяем 4 эффекта.

### 3.1. Bloom (свечение ярких областей)

**`extractBright(uv)`** — выделяет только яркие пиксели:
```glsl
float luminance = dot(c, vec3(0.2126, 0.7152, 0.0722));  // формула sRGB luminance
float mask = smoothstep(0.62, 1.0, luminance);            // мягкий порог
return c * mask;
```
- `0.2126·R + 0.7152·G + 0.0722·B` — стандартная формула яркости в линейном sRGB (зелёный весит больше всего, потому что глаз чувствительнее к нему).
- `smoothstep(0.62, 1.0, lum)` — плавный порог: совсем тёмные → 0, яркие → 1, между — мягко.

**Дешёвая «размытая» свёртка** — берём яркость в нескольких соседних точках и складываем с весами:
```
bloom = bright(uv) * 0.24
      + bright(uv ± 1.5texel по X) * 0.15
      + bright(uv ± 1.5texel по Y) * 0.15
      + bright(uv ± 3.0texel по диагоналям) * 0.08
```
9 сэмплов на пиксель — это сильно упрощённая замена настоящему gauss blur (который требовал бы либо 25+ сэмплов, либо два прохода — separable). Получаем размытую карту ярких областей.

**Финальное наложение:**
```glsl
color = base + bloom * uBloomStrength;
```
Аддитивный вклад — яркие области «расплываются» на соседние пиксели, создаётся свечение.

---

### 3.2. Vignette (виньетка)

```glsl
vec2 centeredUv = vUV * 2.0 - 1.0;          // UV → [-1, 1] относительно центра
float dist = length(centeredUv);
float vignette = 1.0 - smoothstep(0.45, 1.12, dist);
color *= mix(1.0, vignette, uVignetteStrength);
```
- В центре `dist=0` → `vignette = 1` (без затемнения).
- На краях `dist≈1` → `vignette ≈ 0` (тёмное затемнение).
- `mix(1.0, vignette, strength)` — управляет интенсивностью эффекта.

Имитирует оптический недостаток объектива — края кадра темнее центра. Кинематографично.

---

### 3.3. Grain (киношное зерно)

```glsl
float randomNoise(vec2 p) {
  return fract(sin(dot(p, vec2(12.9898, 78.233))) * 43758.5453);
}

vec2 pixel = vUV / uTexelSize;                      // координата в пикселях
float grain = randomNoise(pixel + vec2(uTime*60, uTime*23)) - 0.5;
color += grain * uGrainStrength;
```
Классический GLSL-хак для псевдо-рандома: `fract(sin(...) * largeNum)`. Никаких генераторов и таблиц — просто чудовищно нелинейная функция координаты, которая воспринимается как шум.

`uTime` каждый кадр сдвигает паттерн → шум **анимируется** (статичный grain выглядит как «грязь на стекле», движущийся — как плёнка).

`- 0.5` центрирует диапазон: шум добавляет/вычитает яркость в обе стороны.

---

### 3.4. Color Grading через 3D LUT

Самая хитрая часть. **LUT (Look-Up Table)** — заранее посчитанная таблица «исходный цвет → итоговый цвет». Это способ применять любую сложную цветокоррекцию за **3 чтения текстуры** (вместо реализации алгоритма в шейдере).

**Проблема**: WebGL2 поддерживает 3D-текстуры, но универсальнее хранить LUT как **strip-текстуру** (2D-текстура размера `size × (size·size)`). Каждый «слой» по синему — отдельная горизонтальная плитка `size × size` с уже окрашенными RGB.

Здесь `size = 33` → размер атласа `33 × 1089`.

**`sampleLutStrip(lutTex, color)`** — основа шейдера:
```glsl
float blueIndex = color.b * (size - 1);
float z0 = floor(blueIndex);                        // нижний слой
float z1 = min(z0 + 1.0, size - 1);                 // верхний слой
float zMix = fract(blueIndex);                      // вес интерполяции

float x0 = (z0 * size + color.r * (size-1) + 0.5) / atlasWidth;
float x1 = (z1 * size + color.r * (size-1) + 0.5) / atlasWidth;
float y  = (color.g * (size-1) + 0.5) / atlasHeight;

vec3 c0 = texture(lutTex, vec2(x0, y)).rgb;        // цвет в нижнем слое
vec3 c1 = texture(lutTex, vec2(x1, y)).rgb;        // цвет в верхнем слое
return mix(c0, c1, zMix);                          // трилинейная интерполяция по Z
```

Что происходит:
1. **Синий канал** определяет, какие два «слоя» атласа взять.
2. **Внутри слоя** красный канал даёт X, зелёный — Y. Bilinear-фильтрация в `texture()` сама интерполирует по R и G.
3. **Между слоями** интерполируем сами по Z (синий) через `mix`.
4. Получается **трилинейная** интерполяция по 3D-кубу — эквивалент честной 3D-LUT.

`+ 0.5` в формулах — сдвиг к центру texel'а, иначе билинейный фильтр захватит соседнюю плитку и получится грязь на границах слоёв.

Итог: `color = mix(color, gradedColor, uGradingStrength)` — плавно перетекает к стилизованной палитре. В этой лабе подгружается `08_Film Emulation LUTs_B&W.cube` → даёт чёрно-белую эмуляцию плёнки.

## 6. `resizeRenderTargets()` (main.js:367) — создание FBO

На каждый resize **пересоздаётся** весь набор:

### `sceneColorTex` — текстура цвета
```js
gl.texImage2D(TEXTURE_2D, 0, RGBA, w, h, 0, RGBA, UNSIGNED_BYTE, null);
```
`null` вместо данных — выделяем место, но не заливаем. Сцена нарисуется сюда.

### `sceneDepthTex` — текстура глубины
```js
gl.texImage2D(TEXTURE_2D, 0, DEPTH_COMPONENT24, w, h, 0, DEPTH_COMPONENT, UNSIGNED_INT, null);
```
24-битная глубина. Фильтр **NEAREST** (для глубины нельзя интерполировать — потеряется точность). Текстура, а не renderbuffer — потому что подготовлено к чтению глубины из шейдера (для Depth of Field, который не дописан).

### Сборка FBO
```js
sceneFbo = gl.createFramebuffer();
gl.bindFramebuffer(FRAMEBUFFER, sceneFbo);
gl.framebufferTexture2D(FRAMEBUFFER, COLOR_ATTACHMENT0, TEXTURE_2D, sceneColorTex, 0);
gl.framebufferTexture2D(FRAMEBUFFER, DEPTH_ATTACHMENT,  TEXTURE_2D, sceneDepthTex, 0);
gl.checkFramebufferStatus(FRAMEBUFFER); // должен быть FRAMEBUFFER_COMPLETE
```
- `COLOR_ATTACHMENT0` — куда писать цвет (`gl_FragColor` / `out vec4 outColor`).
- `DEPTH_ATTACHMENT` — куда писать глубину (z-buffer).
- `checkFramebufferStatus` — обязательная проверка, что комбинация форматов поддерживается драйвером.

## 8. Загрузка LUT (main.js:533–680)

### `createGeneratedLutTexture(size, gradingFn)`
Генерит LUT процедурно. Для каждой тройки `(rx, gy, bz) ∈ [0, size)³` считает `gradingFn(r, g, b)` и упаковывает в strip-текстуру:
- Плитки идут по горизонтали (`x = bz·size + rx`), внутри плитки — `y = gy`.
- При `gradingFn = (r,g,b) => [r,g,b]` получается **identity-LUT** (исходник = выход) — используется как «нейтральный» fallback.

### `createLutTextureFromCubeText(cubeText)`
Парсит **Adobe `.cube` формат** (стандарт индустрии цветокоррекции):
```
LUT_3D_SIZE 33
0.0 0.0 0.0
0.03 0.0 0.0
...
```
- Ищет строку `LUT_3D_SIZE` → размер куба.
- Дальше `size³` строк по 3 float — RGB-значения, идущие в порядке `bIndex` быстрее всего, потом `gIndex`, потом `rIndex` (правило Adobe Cube).
- Раскладывает их в strip-текстуру той же раскладки, что и `createGeneratedLutTexture`.

Файл `textures/08_Film Emulation LUTs_B&W.cube` загружается через `fetch` асинхронно. До загрузки используется identity-LUT.

## 11. Главный `render()` (main.js:849) — оркестрация

**Ключевая развилка:**

```js
if (bloomEnabled || vignetteEnabled || grainEnabled || gradingEnabled) {
  // === ПРОХОД 1: рисуем в FBO ===
  gl.bindFramebuffer(FRAMEBUFFER, sceneFbo);     // переключаем выход на FBO
  gl.clear(COLOR | DEPTH);
  drawScene();                                    // сцена → sceneColorTex + sceneDepthTex

  // === ПРОХОД 2: пост на экран ===
  gl.bindFramebuffer(FRAMEBUFFER, null);          // обратно на экран
  gl.clear(COLOR);
  gl.disable(DEPTH_TEST);                         // постшейдеру не нужна глубина

  gl.useProgram(postProgram);
  gl.activeTexture(TEXTURE0); gl.bindTexture(TEXTURE_2D, sceneColorTex);
  gl.activeTexture(TEXTURE1); gl.bindTexture(TEXTURE_2D, lutIdentityTex);
  gl.activeTexture(TEXTURE2); gl.bindTexture(TEXTURE_2D, lutStyledTex);
  // ... uniform'ы силы эффектов ...
  gl.bindVertexArray(postVao);
  gl.drawArrays(TRIANGLES, 0, 6);                 // полноэкранный квад
  gl.enable(DEPTH_TEST);
} else {
  // === Сразу на экран без пост ===
  gl.bindFramebuffer(FRAMEBUFFER, null);
  drawScene();
}
```

**Анимация сил эффектов:**
```js
const anim = 0.5 + 0.5 * Math.sin(t * 0.9);     // 0..1, пульсация
bloomStrength    = 0.85 * anim
vignetteStrength = 0.6 + 0.6 * anim
grainStrength    = 0.14 * (0.3 + 1.2 * anim)
gradingStrength  = 1.0 * anim
```
Все 4 эффекта «дышат» по времени — это закрывает пункт задания «анимация параметров».

## Что точно спросят на защите

| Вопрос | Ответ |
|---|---|
| Что такое framebuffer? | «Холст» для рендера. Default — экран. Свой FBO позволяет рисовать в текстуру, а не на экран. Базовая техника постобработки, отражений, теней. |
| Зачем off-screen рендер для постобработки? | Чтобы получить «снимок сцены» как текстуру и обработать каждый её пиксель шейдером. Без FBO нельзя прочитать пиксели, которые сам же только что нарисовал. |
| Что такое полноэкранный квад? | 2 треугольника в clip-space `[-1,1]²`, покрывающие весь экран. Используется как «холст» для постшейдера — каждый пиксель экрана прогоняется через фрагмент один раз. |
| Что такое COLOR_ATTACHMENT и DEPTH_ATTACHMENT? | Точки прикрепления текстур к FBO. COLOR_ATTACHMENT0 — куда писать `outColor`, DEPTH_ATTACHMENT — куда писать z. Можно прикреплять и render-buffer (если читать не нужно), но здесь текстура (для будущего DoF). |
| Что делает `checkFramebufferStatus`? | Проверяет, что комбинация форматов цвета и глубины поддерживается драйвером. Если вернёт что-то кроме `FRAMEBUFFER_COMPLETE` — FBO нерабочий. |
| Как работает Bloom? | Выделяем яркие пиксели по luminance с мягким порогом → размываем (свёртка по нескольким соседям) → аддитивно складываем с базой. Имитирует растекание света в оптике. |
| Что такое luminance и зачем веса 0.2126/0.7152/0.0722? | Это веса каналов R/G/B для расчёта воспринимаемой яркости в sRGB. Глаз сильнее всего реагирует на зелёный, слабее всего — на синий. Это стандарт ITU-R BT.709. |
| Что такое 3D LUT и как он работает? | Заранее посчитанная таблица «входной цвет → выходной цвет» для всех комбинаций RGB. Применение = 3 чтения текстуры + интерполяция. Любая сложная цветокоррекция за одну и ту же стоимость. |
| Как 3D-LUT хранится в 2D-текстуре? | Strip-атлас: `size × (size·size)`. Каждый слой по синему — отдельная горизонтальная плитка `size × size`. Чтение: bilinear-фильтр сам интерполирует по R и G, по Z (синий) интерполируем вручную через `mix(c0, c1, zMix)`. |
| Зачем `+ 0.5` в формулах LUT-координат? | Сдвиг к центру texel'а. Иначе билинейный фильтр захватит соседнюю плитку на границе и получится мусор. |
| Что такое `.cube` формат? | Adobe Cube LUT — текстовый формат для 3D-LUT. `LUT_3D_SIZE N` + `N³` строк по 3 float (RGB). Стандарт в DaVinci Resolve, Premiere, After Effects. |
| Как работает Grain? | Псевдо-рандом `fract(sin(dot(p, ...)) * largeNum)` по координате пикселя + времени. `- 0.5` центрирует, прибавляется к цвету. Анимация через `uTime`. |
| Как Vignette затемняет края? | `dist = length(UV*2 - 1)` (0 в центре, ≈1 по краям) → `smoothstep` даёт плавное затемнение → умножается на цвет. |
| Почему `disable(DEPTH_TEST)` перед пост-проходом? | Полноэкранный квад покрывает всё, никакой глубины не нужно, тест только мешал бы. После пост-прохода обратно включаем для следующего кадра. |
| Что будет, если не пересоздавать FBO при resize? | Текстура останется старого размера, при рендере в неё попадёт только часть экрана (либо растянется), при семплировании из неё в постшейдере — кривые UV. |
| Почему сцена в FBO рисуется в `RGBA8`, а не в `RGBA16F`? | Простота и универсальность. `RGBA8` ограничивает динамический диапазон `[0,1]`, что не позволяет настоящий HDR bloom (где надо знать «насколько ярче 1.0 был пиксель»). В этой лабе bloom приближённый. |
| Сколько проходов рендера в кадре? | Два: 1) сцена → FBO. 2) FBO как текстура → полноэкранный квад на screen. Если эффекты выключены — один проход прямо на screen. |
